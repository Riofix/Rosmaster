# STM32 底盘运动控制固件 (STM32F103)

该固件主要负责上位机（如 RDK X5 / ROS / PC）与下位机驱动器之间的数据桥接、协议解析与调度控制，使用基于无锁环形队列和表驱动分发的架构。

## 通信架构流向
上位机指令 ➔ **(USART2 RX/DMA)** ➔ **协议解析状态机 (Protocol)** ➔ **无锁环形队列 (MsgQueue)** ➔ **主轮询分发引擎 (App_Protocol_Tick)** ➔ **硬件驱动执行**。

---

## 串口通信协议格式

固件与上位机之间的通信严格遵守以下帧格式：

| 帧头 (Header) | 命令号 (CMD) | 长度 (LEN) | 数据 (DATA) | 校验和 (CHECKSUM) |
|:---:|:---:|:---:|:---:|:---:|
| 0xFF / 0xFB | 1 Byte | 1 Byte | N Byte | 1 Byte |

*   **接收包头 (RX Header)**：`0xFF` (上位机 -> STM32)
*   **发送包头 (TX Header)**：`0xFB` (STM32 -> 上位机)
*   **长度 (LEN)**：仅指 `DATA` 字段的字节数。
*   **校验和 (CHECKSUM)**：`Header + CMD + LEN + DATA[0] + ... + DATA[N]` 的低8位累加和。

## 指令集与数据格式映射表

### 1. 步进电机控制 (EMM_V5) - 指令区 0x50~0x5C
> DATA 字段的第0字节均为目标电机地址 `addr`。

| CMD | 功能描述 | DATA 格式 | 示例 (Hex, addr=1) |
| :---: | :--- | :--- | :--- |
| **0x50** | 脱机与使能控制 | `[addr][state]` (0:脱机, 1:使能) | `FF 50 02 01 01 54` |
| **0x51** | 速度模式控制 | `[addr][dir][vel_H][vel_L][acc]` | `FF 51 05 01 01 01 F4 00 4D` (500RPM) |
| **0x52** | 位置模式控制 | `[addr][dir][vel_H][vel_L][acc][pulse_32bit][rel]` | `FF 52 0A 01 01 01 F4 00 ...` |
| **0x53** | 紧急停止运动 | `[addr]` | `FF 53 01 01 55` |
| **0x54** | 触发同步开始 | `[addr]` | `FF 54 01 01 56` |
| **0x55** | 设置单圈零点 | `[addr][loop]` (0:单圈, 1:多圈) | `FF 55 02 01 00 58` |
| **0x57** | 触发回零动作 | `[addr][type]` | `FF 57 02 01 01 5B` |
| **0x5A** | 重置当前位置 | `[addr]` | `FF 5A 01 01 5C` |
| **0x5C** | 读取系统参数 | `[addr][param_type]` | `FF 5C 02 01 01 60` |

### 2. 舵机与无刷电机 (PWM) - 指令区 0x70~0x74

| CMD | 功能描述 | DATA 格式 | 参数说明 |
| :---: | :--- | :--- | :--- |
| **0x70** | 舵机角度控制 | `[ch][angle]` | ch: 1=PB0, 2=PB1; angle: 0~180 |
| **0x71** | 舵机脉冲控制 | `[ch][pulse_H][pulse_L]` | pulse: 500~2500 (us) 微秒 |
| **0x72** | 无刷调速(占空比) | `[duty]` | duty: 0~100 (%), 固定PA6输出 |
| **0x73** | 无刷调速(脉冲) | `[pulse_H][pulse_L]` | pulse: 1000~2000 (us), 固定PA6 |
| **0x74** | 无刷电机急停 | (无数据) | 仅停止无刷电机，舵机保持原位 |

### 3. MPU6050 姿态传感器 - 指令区 0x75~0x78

| CMD | 功能描述 | 说明 |
| :---: | :--- | :--- |
| **0x75** | 单次读取姿态 | 立即返回一帧 0x64 姿态数据 |
| **0x76** | 陀螺零偏校准 | 静态放置时发送，需约0.4秒，完成后返回应答 |
| **0x77** | 开启自动流上报 | STM32 每 2ms 主动发送一帧 0x64 数据 |
| **0x78** | 关闭自动流上报 | 停止主动上报 |

### 4. STM32 -> 上位机 反馈 (TX) - 指令区 0x60~0x64
> 帧头固定为 `0xFB`。

| CMD | 功能描述 | DATA 格式 |
| :---: | :--- | :--- |
| **0x60** | 查询参数返回 | `[addr][params...]` (根据查询类型变长) |
| **0x61** | 成功应答 (ACK) | `[addr]` (确认指令已接收并执行) |
| **0x62** | 错误报警 (ERR) | `[addr][err_code]` (1:Flash, 2:引数错, 3:未知CMD) |
| **0x64** | **姿态数据流** | `[Roll_float][Pitch_float][Yaw_float]` (12字节, 均为小端float) |

## 测试示例
*   **读取 MPU 数据**：发送 `FF 75 00 74`
*   **开启 MPU 自动流**：发送 `FF 77 00 76`
*   **舵机(PB0)转到90度**：发送 `FF 70 02 01 5A CC`
*   **无刷电机(PA6) 50%油门**：发送 `FF 72 01 32 A4`
*   **停止无刷电机**：发送 `FF 74 00 73`
*   **电机1(addr=1) 500RPM**：发送 `FF 51 05 01 01 01 F4 00 4C`

---

## 工程特性
1.  **高并发性**：基于 SPSC 无锁环形队列，中断与主循环解耦。
2.  **易扩展性**：采用 PWM 硬件描述符表，增加 PWM 引脚无需改动驱动核心逻辑。
3.  **稳定性**：加入超时帧自动抛弃机制。

# 🔩 STM32F103C8T6 下位机固件

**2026年中国大学生机械工程创新创意大赛 —— 智能机器人底层控制固件**

本固件运行于 **STM32F103C8T6 (ARM Cortex-M3)** 微控制器，作为机器人的下位机，负责与上位机 (RDK X5 / ROS 2) 通信并驱动机器人硬件。

## 🏗️ 软件架构

固件采用 **三层架构** 设计，实现硬件驱动、协议解析与应用逻辑的完全解耦。

### 1. 🧩 协议层 — `Protocol/`

**核心**: `protocol.c` + `msg_queue.c`

上位机指令 ➔ **(USART2 RX/DMA)** ➔ **协议解析状态机** ➔ **无锁环形队列 (MsgQueue)** ➔ **主轮询分发引擎**

| 文件 | 功能 |
|:---|:---|
| `protocol.c/h` | 核心协议帧解析 (0xFF 0xFB/0xFC)、校验和验证、封包发送 |
| `protocol_emm.c/h` | 步进电机 (EMM-V5) 协议扩展指令处理 |
| `cmd_handle.c/h` | 命令分发器：表驱动方式将解析后的指令路由到对应 handler |
| `msg_queue.c/h` | **单生产者-单消费者 (SPSC) 无锁环形队列**，实现中断与主循环的解耦 |

### 2. 📱 应用层 — `APP/`

| 文件 | 功能 |
|:---|:---|
| `main.c` | 系统初始化：SysTick、串口、OLED、ESP8266 配网连接 |
| `app.c/h` | 主轮询调度引擎 `App_Protocol_Tick()`，循环分发并执行协议指令 |
| `app_bldc.c/h` | 无刷电机 PWM 调速控制 |
| `app_imu.c/h` | MPU6050 姿态解算：集成 **Mahony 四元数滤波器**，输出 Roll/Pitch/Yaw |
| `app_motor.c/h` | 步进电机运动控制 (位置/速度/回零/同步) |
| `app_rgb.c/h` | RGB LED 颜色控制 |
| `app_servo.c/h` | 舵机角度与脉冲宽度控制 |

### 3. 🔌 板级支持包 — `BSP/`

| 文件 | 外设 | 接口 |
|:---|:---|:---:|
| `bsp_esp8266.c/h` | ESP8266 WiFi 模块 | USART + AT 指令 |
| `bsp_iic.c/h` | 软件 I2C | GPIO 模拟 |
| `bsp_mpu6050.c/h` | MPU6050 六轴姿态传感器 | I2C |
| `bsp_pwm.c/h` | PWM 输出 | TIM 定时器 |
| `bsp_systick.c/h` | SysTick 精准延时 | SysTick |

## 🔌 串口通信协议格式 (V2.0)

固件与上位机之间的通信严格遵守以下帧格式，支持双包头识别与自定义校验。

### 帧结构

| 帧头1 | 帧头2 | 长度 (LEN) | 数据 (DATA) | 校验和 (CS) |
|:---:|:---:|:---:|:---:|:---:|
| 0xFF | 0xFB/0xFC | 1 Byte | N Byte | 1 Byte |

### 协议规则

| 项目 | 说明 |
|:---|:---|
| **接收包头 (RX Header)** | `0xFF 0xFB` (上位机 → STM32，下行) |
| **发送包头 (TX Header)** | `0xFF 0xFC` (STM32 → 上位机，上行) |
| **长度 (LEN)** | 包头之后的所有字节数。计算: `LEN = 1 + N + 1` (LEN位 + 数据位 + 校验位) |
| **校验和 (CS)** | 从 LEN 字段开始累加，到校验位前的最后一个字节，结果对 256 取余 |
| **字节序** | **小端模式 (Little-Endian)**，多字节数据低位在前 |

## 📋 指令集与数据格式映射表

### 1. 步进电机控制 (EMM_V5) — 指令区 `0x50~0x5C`
> DATA 字段的第0字节均为命令字 (CMD)，第1字节为电机地址 `addr`。

| CMD | 功能描述 | DATA 格式 |
| :---: | :--- | :--- |
| **0x50** | 脱机与使能控制 | `[CMD][addr][state]` (0:脱机, 1:使能) |
| **0x51** | 速度模式控制 | `[CMD][addr][dir][vel_L][vel_H][acc]` |
| **0x52** | 位置模式控制 | `[CMD][addr][dir][vel_L][vel_H][acc][clk_0~3][rel]` |
| **0x53** | 紧急停止运动 | `[CMD][addr]` |
| **0x54** | 触发同步开始 | `[CMD][addr]` |
| **0x55** | 设置单圈零点 | `[CMD][addr][loop]` |
| **0x57** | 触发回零动作 | `[CMD][addr][type]` |
| **0x5A** | 重置当前位置 | `[CMD][addr]` |
| **0x5C** | 读取系统参数 | `[CMD][addr][param_type]` |

### 2. 舵机与无刷电机 (PWM) — 指令区 `0x70~0x74`

| CMD | 功能描述 | DATA 格式 |
| :---: | :--- | :--- |
| **0x70** | 舵机角度控制 | `[CMD][ch][angle]` |
| **0x71** | 舵机脉冲控制 | `[CMD][ch][pulse_L][pulse_H]` (500~2500us) |
| **0x72** | 无刷调速(占空比) | `[CMD][duty]` (10~99%) |
| **0x73** | 无刷调速(脉冲) | `[CMD][pulse_L][pulse_H]` |

### 3. MPU6050 姿态传感器 — 指令区 `0x75~0x78`
> **解算方式**: 采用四元数 (Mahony 滤波) 计算欧拉角，消除万向节锁。

| CMD | 功能描述 | 说明 |
| :---: | :--- | :--- |
| **0x75** | 单次读取姿态 | 返回 0x64 姿态数据包 |
| **0x76** | 陀螺零偏校准 | 静态放置，耗时约 0.4s |
| **0x77** | 开启自动上报 | 2ms 周期主动发送 0x64 数据 |
| **0x78** | 关闭自动上报 | 停止主动发送 |
| **0x79** | 里程计查询 | 返回 pos, target, mode, raw_ticks |

### 4. STM32 → 上位机 反馈 (TX) — 指令区 `0x61~0x65`
> 帧头固定为 `0xFF 0xFC`。所有多字节数值均为**小端序**。

| CMD | 功能描述 | DATA 格式 |
| :---: | :--- | :--- |
| **0x61** | 成功应答 (ACK) | `[CMD][addr]` |
| **0x62** | 错误报警 (ERR) | `[CMD][addr][err_code]` |
| **0x64** | **姿态数据流** | `[CMD][Roll(f)][Pitch(f)][Yaw(f)]` (Float, 13字节) |
| **0x65** | **里程计数据** | `[CMD][pos(f)][target(f)][mode][ticks(i32)]` (14字节) |

## 🏆 工程特性

1. **高并发性** — 基于 SPSC 无锁环形队列，串口接收中断与业务逻辑分发完全解耦，无锁无竞争
2. **姿态解算** — 集成 **Mahony 四元数滤波器**，核心算法高频更新，有效克服万向节锁与短路噪声
3. **对齐一致性** — 全链路采用**小端字节序 (Little-Endian)**，极大降低了上位机 (RDK X5 / ROS 2) 的解析负担
4. **稳定性** — 集成串口超时状态机重置与校验错误计数器，自动恢复异常通信状态
5. **表驱动分发** — 命令处理采用查表法，新增指令只需注册回调，无需修改框架代码
6. **WiFi 桥接** — 通过 ESP8266 实现 TCP 透传，支持三个抓手的远程控制

## 🚀 开发与烧录

### 开发环境
- **IDE**: Keil MDK (μVision V5)
- **编译器**: ARMCC / AC6
- **调试器**: J-Link / ST-Link
- **芯片**: STM32F103C8T6 (72MHz, 64KB Flash, 20KB RAM)

### 烧录步骤

1. 打开 Keil MDK，加载 `MDK-ARM/` 目录下的工程文件
2. 在 `main.c` 中配置 WiFi 参数:
   ```c
   // 修改为你的热点信息
   #define WIFI_SSID     "Digua"
   #define WIFI_PASSWORD "12345678"
   #define SERVER_IP     "192.168.1.1"
   #define SERVER_PORT   8080
   ```
3. 编译 (F7) → 下载 (F8)
4. 观察 OLED 显示屏确认 WiFi 连接状态

> **注意**: 如果 ESP8266 配置失败，OLED 会显示错误代码，请检查 AT 固件版本和网络配置。
